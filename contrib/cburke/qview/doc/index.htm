qview

Introduction

qview is a grid and plot system for displaying kdb+ tables in a linux console session. It uses gtk cairo graphics, and is built as a linux shared object.

The plot styles are similar to those in contrib/cburke/amcharts. The amcharts plots are more professional looking but much slower to load and better suited for static display, while qview plots can be updated in real time.

To install, run make in the source directory (glib and gtk development libraries are required).

Define cover functions as: 

grid:`qview 2:(`grid;1)
plot:`qview 2:(`plot;1)
grids:`qview 2:(`grids;1)
plots:`qview 2:(`plots;1)

Syntax

The argument is either a table, or a pair (table;parameter list). Parameters are a list of (keyword;value...), for keywords caption, title, window. For example: 

grid tab      / grid tab in default window (0)
plots tab     / plot tab in new window
plot(tab;("window";3;"title";"myplot"))   / plot tab in window 3, with given title

Windows

By default, grid and plot reuse a single window (window 0), while grids and plots create a new window. Each function returns its window handle.

Window handles are from 0 to 99.

Grid

Grid has a single style.

Plot types

The table meta data indicates the type of plot. A Price plot is drawn if other types do not match.

Book Depth
Open/High/Low/Close
Price
RegNMS
Volume Profile

Demos

The demo directory has examples of each plot. Copy to your q home directory, then load q and: 

\l demo/qview.q
