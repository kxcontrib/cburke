Snapshots

Project Snapshots

These are intended as simple way of allowing comparisons between the current state of the project and an earlier state, or recovering from an earlier version. They are not a substitute for a proper version control system like Git or Subversion. Instead, snapshots are complementary, intended primarily for quick comparisons of work in progress, rather than keeping track of major code releases.

The number of Snapshots kept is given in base.cfg, default 5.

Snapshots are made automatically, and can also be made manually. Automatic snapshots are made when a project is loaded, as follows:

  • if the snapshot directory does not exist or is empty, make a snapshot and exit.
  • if there is a snapshot with today's date, then exit.
  • if the current contents of the project match the most recent snapshot, then exit.
  • make a snapshot.

Manual snapshots can be made by selecting menu Project|Make Snapshot. Snapshots are made only if the current project directory differs from the most recent snapshot.

To compare project snapshots, select menu Project|Project Snapshots. This runs Directory Match on the snapshots and current version. By default, the latest snapshot is compared with the current project.

Script Snapshots

Script snapshots are automatically saved as scripts are changed and run. Snapshots are saved for changes today and the most recent date with any change in the project.

Use menu Script|Script Snapshots to view, compare and restore the snapshots.

Snapshots are stored in the snap/.snp subdirectory.

Comparison Utility

Snapshot comparisons can use a basic built-in viewer, but for best use, define an external viewer in XDiff in base.cfg.
