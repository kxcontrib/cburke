Edit

This is a tabbed text editor, with an optional sidebar, project manager, code snapshots, and various run options for scripts.

Press Ctrl+M in the terminal to show the editor. In the editor, press Ctrl+B to hide/unhide the sidebar and Shift+Ctrl+M to show the second edit window.

Script tabs can be rearranged by drag and drop.

Scripts are saved automatically before any run, and on close.

Color and syntax highlighting is supported for q and k, as well as other languages. The color config is in lib/edit.xml, and syntax coloring in k.lang and q.lang.

Run Options

In the following:

  • A line includes subsequent lines if part of a multi-line sentence, see Multi Line.
  • some commands include show result, meaning that an assigned result is also displayed in the terminal

Ctrl+Enter - run line at cursor, do not advance cursor (allows repeated run of line).

Shift+Ctrl+Enter - as Ctrl+Enter, plus show result.

Ctrl+R - run line at cursor, and advance to next line.

Shift+Ctrl+R - as Ctrl+R, plus show result.

Ctrl+E - run selected lines.

Shift+Ctrl+E - run all lines. Comment blocks are ignored - a line with only / starts a comment block, and only \ finishes a comment block. Outside of a comment block, a line with only \ terminates the run. Note that Shift+Ctrl+T clears the terminal, so it can be useful to hold down the Shift+Ctrl keys, then press T and E - this clears the output terminal and then runs all lines into the cleared terminal.

Ctrl+L - load file into the session.

F8 - run the clipboard.

F9 and Ctrl+F9 load the project build/run scripts, see Folders and Projects.

Multi Line

qconsole reads multi-line entries from the edit window, as does the interpreter when loading a script. Any line that is indented is assumed to be part of the previous command. Thus the following is a single input line in the edit window:

show 10#s:

 h"select sum size by date,5 xbar time.minute from trade
   where date in -30 sublist date,sym=`IBM"

Searching

Ctrl+F is Find/Replace in the current script and Shift+Ctrl+F is Find in Files.

Ctrl+click on a name will search for that name in the current project directory. If a single match is found, the corresponding script is opened at the match. If more than one match is found, the Find in Files dialog is opened with the several matches. If not found, the text selected is simply highlighted as usual.

Git

If the source is under git version control, the Project menu includes options to run gitgui and show git status.
